## 🏗️ 什么是 OrangeSea?

OrangeSea 是一个建立在现有互联网之上的**覆盖网络 (Overlay Network / Chat-VPN)**。它执行“拒绝搭便车”策略，以确保网络的韧性。

### 1. 统一客户端与强制贡献
- **One App, All Nodes:** 不区分服务器和客户端。
- **VPN 即服务:** 移动端 (iOS/Android) 启动即申请系统级 VPN 权限，成为网络节点。
- **责任滑块 (The Slider of Duty):** 用户无法关闭上传贡献，只能调节档位：
  - 🟢 **节能 (Eco):** 最低心跳包。无 Token 奖励。
  - 🔴 **狂暴 (Overdrive):** 最大带宽贡献。最大化 Token 奖励。

### 2. 双轨制经济模型 (核心创新)
为了防止资本垄断（抗女巫攻击），我们严格将 **“利”** 与 **“名”** 分离。

| 维度 | **Tinder Token (火种币)** | **Reputation Score (信誉分)** |
| :--- | :--- | :--- |
| **来源** | 物理层贡献 (带宽/存储) | 社交行为 & 履约记录 |
| **可交易?** | ✅ 是 | ❌ **否 (灵魂绑定)** |
| **用途** | 支付、兑换 | 治理权重、**签证申请** |

### 3. 抗女巫攻击：信贷爬坡 (The Credit Ramp)
基于银行风控逻辑：
- **白户限制:** 新节点（0信誉）的 Token 产出系数 ≈ 0。
- **社交证明:** 要想挖矿赚钱，节点必须通过 **信任网络 (Web of Trust)** 建立连接，证明自己是真人。
- **结果:** 使得机器脚本刷号在经济上无利可图。

## 🛠️ 技术栈目标

我们需要以下领域的专长：

- **核心层:** Rust (Libp2p, Tokio, QUIC)。
- **移动网络:** iOS `NetworkExtension` (PacketTunnelProvider), Android `VpnService`。
- **节点发现:** DHT (Kademlia), MDNS, 蓝牙/Wi-Fi Direct (近场通讯)。
- **隐私层:** ZK-SNARKs (用于身份/信誉证明)。
